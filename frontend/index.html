How It Works

1
Return-Likelihood Classifier
LightGBM trained on temporal split (2009–H1 2011 train, H2 2011 test). Features: customer behavioral history + transaction context. Output: P(return) per transaction.
2
Excessive-Returner Anomaly Detection
Isolation Forest on customer-level behavioral features — no labels required. Validates against top-decile return-value-ratio heuristic.
3
Customer Segmentation
KMeans k=4 on RFM + return features. Segments: Premium Loyal · Healthy Browser · At-Risk · Returner. Enables differentiated return policy.
4
Substitute Recommender
Hybrid content (sentence-transformers embeddings) + CF (implicit ALS). NDCG@10 evaluation. Turns return moments into retention opportunities.
Dataset: UCI Online Retail II · 1,067,371 transactions · Dec 2009–Dec 2011 · Source code
